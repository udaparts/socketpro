Major events about SocketPro evolution history:

18.    04/2020-present -- Release SocketPro versions 6.3.1.1-8 with a new feature Interrupt:

  • Core libraries are added with a new feature Interrupt, which could be used to interrupt a looping processing at the other end.
  • All adapters are updated with the new feature Interrupt.
  • Make MySQL 8 server plugin support its version 8.0.19 or later.
  • Add support to char16_t.
  • Recompile all libraries after adding support of char16_t.
  • Fix a window server core bug related with random sequencing.
  • C++ Samples and tutorials updated because of adding support of char16_t.
  • Add support of streaming downloading of multiple files for best performance over small files.
  • Improve both window and linux crash information output for better debugging.
  • Use -fvisibility=hidden to reduce exported symbols of Linux shared libraries.
  • Fix the method Cancel bugs with secure socket and message queue.
  • Fix a server routing error related with client message queue enabled.
  • Fix a server side crash within libuservercore.so when publishing a message from web browsers.

17.    06/2019-03/2020 -- Release SocketPro versions 6.3.0.1-10 with a few improvements and bug fixes:

  • Add the method PostProcessing to dispatch requests from calling thread onto pool worker thread within client core library.
  • Re-write remote file service to use PostProcessing instead.
  • Fix the issue, which the virtual function OnReleaseResource(bool bClosing, uint info) at server adapter level, was not called when a session is closed.
  • Performance tuning with improvement about 6% overall with core libiraries, usocket and uservercore.
  • Remove dependency over boost libraries within C/C++ development environment with Linux platforms.
  • Add support of enterprise license.
  • Fix a connection time-out issue within window client core library.
  • Fix MySQL 8.0.x server plugin crash issue when closing a session after opening a session to MySQL 8.0.x database.
  • Recompile core libraries (both client and server core libraries) using Visual C++ 2017 instead of old Visual C++ 2010.
  • SocketPro supports windows ce/device 4.0 and windows vsta or later by default.
  • SocketPro still supports windows XP and 2003 by old Visual C++ 2010 compilation per your requirement.
  • Other minor improvements with adapters.
  • Use spin lock extensively within C/C++ adapter whenever it is proper.
  • Fix the issue that a pool of client sockets may fail to connect to a remote server under extremely situations.
  • Use one set of core libraries (usocket and uservercore) for all linux variants.

16.    12/2018-05/2019 -- Release SocketPro versions 6.2.0.1-4 with a few improvements and bug fixes:

  • Add support to TLSv1.2, and disable old protocol SSL3.
  • Add support to TLSv1.3 on Linux variants, and make sure TLSv1.3 is able to communicate with TLSv1.0, 1.1 and 1.2.
  • Implement SocketPro adapter for PHP.
  • Create unit test codes and samples for PHP adapter.
  • Remove a potential dead-lock related with routing at server side when there are many socket disconnections at the same time.
  • Fix a potential crash related socket disconnection at client side.
  • Fix a potential dead-lock at server side when SSL/TLS handshake is done on Linux variant platforms.
  • Improve node.js adapter for better DB query performance.
  • Other minor improvements.

15.    12/2017-11/2018 -- Release SocketPro versions 6.1.0.1-5:

  • Add new feature into SocketPro core libraries for automatic ordering on random returning requests.
  • Improve fault-tolerance, no message loss and connection auto recovery.
  • Use delegate DDiscarded to replace the old DCancel.
  • Upgrade adapters to support the above changes.
  • Add examples for demonstration of automatic ordering on random returning requests, fault-tolerance and connection auto recovery.
  • Make file streaming service free to the public.
  • Implement Mariadb SQL-streaming database plugin and make it free to the public.
  • Implement MySQL/Mariadb SocketPro SQL-streaming server plugin and make it free to the public.
  • Implement SocketPro Node.js adapter.

14.    12/2016-11/2017 -- Release SocketPro versions 6.0.3.1-6.0.4.7:

  • Add persistent queue manual batch support at server side.
  • Improve server dequeuing performance especially for small messages at server side.
  • Add or upgrade SocketPro documentations.
  • Fix memory leaking within JAVA JNI library juclient.
  • Fix a bug of CUQueue move operator of C++ adapter.
  • Fix server router auto-recovery (fault-tolerance) issue at client side.
  • Slightly improve small message server writing performance.
  • Apply SocketPro continuous in-line request/result batching onto SQLite, MySQL, SQL server and ODBC for SQL streaming processing and returning.
  • Fix a potential WaitAll dead lock within client core library when calling the method from two different threads against the same socket at client side.
  • Fix a crash bug within client core library when locking a socket from a socket pool at client side.
  • Fix a potential application ending crash bug on window platforms when killing an application at end.
  • SocketPro adapters upgraded with new feature master/slave pools for enterprise applications and their demo examples. These master pools also support real-time update caches for reduction of network round trips between middle/front tier and back-end relational database or other types of data sources.
  • SQLite streaming plug-in insert, update and delete events are implemented on triggers instead of hooks so that it is the same as MySQL plug-in implementation.

13.    12/2015-11/2016 -- Release SocketPro versions 6.0.2.12-24:

  • Don't use openssl any more. Instead, both client and server cores use MS secure channel to SSL/TLSv secure communication on window platforms.
  • Now, server core library supports a pool of threads for main threads. This is new feature to improve server performance for ssl, online zipping/unzipping and HTTP/websocket parsing.
  • Client side is added with client queue auto merge feature, which enables client auto fault recovery through a pool of sockets that can be connected with different server applications.
  • Improve dequeue performance against high volume of small messages.
  • SocketPro server core libraries are added with four new message chat functions, SpeakPush, SpeakExPush, SendUserMessagePush and SendUserMessageExPush.
  • SocketPro adapters are updated with async databases (SQLite and MySQL) supported.
  • Make async sqlite and mysql/mariadb services for free.
  • Fix server crash when calling the method StopQueueByHandle or StopQueueByName with the input permanent equal to true.
  • Fix missing auto enqueue notification when doing transaction equeuing at server side.
  • Slightly improve dequeue performance over one message queue file shared among multiple consumers.
  • Fix a potential dead-lock inside client core library when shutting down a pool of sockets.
  • SocketPro adapters are updated to echo the above changes.

12.    08/2015 -- Release SocketPro version 6.0.2.10:

  • Fixed the bug that the structure CMessageSender::IpAddress inside client core library libusocket.so may point to garbage within Java environment on linux platforms, which may lead to crashes.
  • C/C++ header files updated to support Android C/C++ compilation.
  • Add method Dispose onto java adapter class CReplication to explicitly destroy its internal socket pool.

11.    05/2015 -- Release SocketPro version 6.0.2.9:

  • improvements on dequeue performance from both client and server core libraries.
  • Fixed message loss problem when dequeued from multiple consumers at server side.
  • Fixed SocketPro java adapter crashes on linux platforms.

10.    04/2015 -- Release SocketPro version 6.0.2.8:

  • Considerable improvements on performance from both client and server core libraries.
  • Minorimprovements on java jni libraries.
  • Removed over locks for multi-threading environments in SocketPro adapters at client side.
  • Update sample qperf (c++, C# and java) for demonstration of queue performance improvements by user batch approach.

9.    03/2015 -- Release SocketPro version 6.0.2.4-7:

  • Message queues are default to oMemoryCached for fast writing.
  • Add structure exception handling to ensure memory cached messages written into queue files before application exits.
  • Improved persistent message queue performance especially on high volume of small messages. Our test results show that SocketPro is easily able to write 400,000 small messages (200 bytes) per second with the same cheap machine. In regards to cross-machine communication plus 1 gbits network, SocketPro is able to easily support 300,000 small messages per second! SocketPro is number one in persistent message queue performance.
  • Add fully supports on async and await for .net and C++ adapters.
  • Java adapter jspa.jar now fully supports java 7/8 SDKs or later. Java tutorial and samples are re-written from java 7 SDK to echo this change.
  • Add fully supports on async and await for .net and C++ adapters.
  • A number minor bugs inside adapters are fixed.

8.    10/2014 -- Release SocketPro version 6.0.2.1-3:

  • SocketPro fully supports python development environments across both windows and linux variant platforms.
  • Performance optimizations at core and adapter levels.
  • Support loading server standard system dynamic libraries created from C/C++.

7.    07/2014 -- Release SocketPro version 6.0.1.2-3:

  • SocketPro starts supporting cross-platform developments on windows and linux variants with new key features, websocket and bi-directional persistent message queue as well as data replication.

  • SocketPro version 5 matured and retired now.
