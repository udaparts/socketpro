Later, you can effortlessly exchange requests and returned results from binary format into original data types compatible with both server and client sides.

  • Tutoral 2: SocketPro secure communication and built-in bi-directional message pushes: First, we need to secure sensitive data movement among computers.   • Tutorial 3: High performance persistent message queue for asynchronous communications among enterprise applications: Requests can be persistently en-queued into a disk data file. If an application is crashed or its network is down, requests can be restored from the disk file after the application is restored and the network is online. As you can see, persistent message queue ensures the reliability of the whole application system. In regards to server persistent message queue, it can be used as asynchronous communication among offline clients through a server or broker. A client sends a message to the server and queues it into a persistent message queue. Afterwards, another client can de-queue the message at its own pace. SocketPro fully supports persistent message queues at client or/and server. This tutorial is focused on server persistent message queue only.
  • Tutorial 4: Network Multiple Computers/Processors for Parallel Computing: SocketPro provides two features for distributing workloads across multiple computing resources. The first feature is called client socket pool, which enables a client to access multiple servers for fast and scalable processing requests asynchronously and in parallel as shown in Tutorial 8: Build responsive and scalable web applications in a few hours. The second feature is called SocketPro server routing or load balancing. Each SocketPro server is able to support multiple routings by different configurations.
  • Tutorial 5: Cross-platform data replication by persistent message queue: SocketPro supports asynchronous transactional replication fully in agreement with all ACID (atomicity, consistency, isolation and durability) properties and guarantees complete consistency among all copies of replicated data. Once target machines are connected, replicate data are pushed over network as early as possible. With the help of SocketPro replication, you can programmatically replicate selected portions of arbitrary data onto different target machines easily.
  • Tutorial 6: SocketPro HTTP service and server push: SocketPro server is written to fully support both HTTP and web socket protocols so that you can access a SocketPro server from web browsers, smart-phone devices (android, iPhone, win phone, etc) and others in case you cannot use SocketPro client components. In addition, SocketPro integrates HTTP/websocket with its own protocol so that you can push any message to anywhere.
  • Tutorial 7: Exchange any size files between client and server across different platforms: It is a common task to write code in order to exchange files between client and server efficiently. SocketPro at its adapter level provides helper utilities for this purpose. In addition, we are going to discuss SocketPro client and server threading models at this tutorial so that you can easily remember calling threads for various events and virtual functions.
  • Tutorial 8: Build resilient, responsive and scalable web applications in a few hours: SocketPro provides a number of unique ways to help build extremely fast response and scalable web applications especially in web server front-tier and middle-tier as well as web browser side.
  • SocketPro performance comparison with window communication framework (WCF) and RabbitMQ: We have created a socket-based framework based on request batch, asynchrony, and parallel computation to accelerate data movement across computers running on different major platforms such as Windows, Linux, and Apple. One of its greatest advantages is that this framework delivers extremely high performance and scalability when compared to other well-known frameworks such as Java RMI, WCF, SOAP/XML web services, etc. This article, along with source codes, demonstrates the performance improvements of SocketPro framework over that of the latest Microsoft technology Window Communication Foundation (WCF) and RabbitMQ. Hopefully, these improvements catch your attention.
  • SocketPro server queue performance study and its comparison with Kafka: SocketPro is written with inline continuous data batching, non-blocking and parallel computation in mind so that networking and CPU processing at both client and server sides are extremely efficient. Under most cases, SocketPro queue is faster than Kafka in both reading and writing middle or large messages having sizes of larger than 200 bytes. Under extremely cases, SocketPro could be about 200% faster than Kafka in both writing and reading messages.
  • Access a SocketPro server anywhere by use of HTTP/WebSocket protocols: SocketPro comes with system client and server core libraries which run on major platforms such as Windows (including Wince and Pocket PC), Linux and Mac. However, SocketPro core libraries, in the near future, will not be ported on various non-major platforms such as IBM mainframe, HP-Unix systems, Sun micro systems, etc. In addition, SocketPro already has a limited number of adapters available for major development environments. However, as you might conceive, SocketPro will not come with adapters for non-major development languages soon, although UDAParts is indeed trying innovative strategies to create new adapters for those languages with high market usage shares. To solve the issues listed above, SocketPro server intentionally plans to create to a full-support web socket that allows the users in the previous situations access a SocketPro server application as long as it is enabled by HTTP/websocket service. We recommend that you use websocket protocol instead of classic HTTP protocol because websocket shows numerous advantages over traditional HTTP protocol, though SocketPro does support HTTP service as well. Therefore, we create this short article to easily guide and quickly allow you to access a SocketPro server application from minor platforms or development languages by use of websocket protocol.
  • Brief introduction of SocketPro continuous SQL-stream sending and processing system (Part 1: SQLite): SocketPro is written to support continuous inline request/result batching and real-time stream processing for the best network efficiency, performance and scalability. UDParts has applied the powerful feature to SQLite, MySQL and ODBC databases, which is named as SocketPro SQL-stream technolgy. This is the first ariticle for SocketPro SQL-stream technology which comes with a set of very simple projects for you to play and study SocketPro SQL-stream technolgy on either windows or Linux platforms. The precompiled libraries on various windows and Linux platforms are totally free to the public. The source code is open to the public for you to study and debug.
  • Brief introduction of SocketPro continuous SQL-stream sending and processing system (Part 2: MySQL): UDAParts has applied SocketPro-stream technolgy on popular MySQL database to create a MySQL server daemon plug-in. You can use the precompiled plugin to significantly accelerate MySQL data accessing with continuous inline request/result batching and real-time stream processing. The performance improvement could be up to 200 and 10 times on WAN and LAN, respectively. You can also rely on the SocketPro MySQL plugin to create a real-time data cache on a middle tier. Further, you can enable other SocketPro services from the MySQL plugin with simple configuration settings without coding. Again, the precompiled libraries on various windows and Linux platforms are totally free to the public. The source code is open to the public for you to study and debug.
