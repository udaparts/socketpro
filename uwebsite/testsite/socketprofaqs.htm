A list of frequent asked questions about SocketPro

General questions

  1. What platforms are supported?

    2. SocketPro has already fully supported the following platforms:

    • Windows XP and its later platforms for both 32bit and 64bit versions

    • Window ce and pocket pc 2003 and their later devices

    • Major 64bit version linux variants such as CentOS/Redhat, Debian, Ubuntu/Mint and OpenSUSE

  2. What core libraries does SocketPro have?

    3. SocketPro comes with two core libraries, usocket and uservercore. The first one runs at client side; and the other at server side. Both are system low-level libraries written from C/C++ for the best performance and scalability with the lowest dependency.

  3. What are SocketPro adapters and why do you neeed these adapters?

    4. SocketPro core libraries are system low-level libraries exposing a considerable number of C functions. There are not friendly to be reused directly from your development environments. In order to reduce development difficulties, UDAParts provides adapters for each of supported development languages. Your codes should directly access one of adapters instead of these C functions. Adapters are created for easy and quick development.

  4. What adapters has SocketPro already come with?

    SocketPro has already come with a number of the following adapters:

    • SocketPro adapter for C++ on window and major Linux platforms

    • SocketPro adapter for .NET development languages (both MS and Mono .NET) on window and linux platforms as well as window ce and pocket pc

    • SocketPro adapter for major web browsers and JavaScript

    • SocketPro adapter for Java versions 1.7 or later

    • SocketPro adapter for Python versions 2.7 or later

    • SocketPro adapter for Node.js versions 6.0.1 or later

    • SocketPro adapter for PHP versions 7.0.1 or later

    Keep in mind that UDAParts will keep on adding more adapters.

  5. My server runs behind a firewall. Can I use SocketPro server within such an environment?

    6. Yes with a little more work! You may need to cooperate with an administrator to turn on a particular port, or do port mapping from a machine to another one. SocketPro is the same as http, https, ftp, and so on in setting up firewall, router and NAT. The setup is really easy with simple clicks.

  6. Can I run HTTP, web socket and SocketPro services at one listening port?

    7. Yes! As shown from all of demo and tutorial server applications with this package, SocketPro server applications can provide multiple services using a single port number by design.

  7. If I have a question or concern to SocketPro, how do I contact UDAParts for technical support?

    Discussion group or email at support@udaparts.com.

  8. Does UDAParts provide customer service?

    Yes! UDAParts does provide consulting services at affordable price. For details, contact UDAParts at support@udaparts.com.

  9. Does SocketPro secure net data communication between a client and a server?

    Yes! SocketPro is using industrial standard SSL3/TLSv1.x libraries to secure communications among machines. Also, SocketPro provides callback at server side for you to write your own code to authenticate a client from its credentials.

Development questions

  1. How do I quickly get started with SocketPro?

    2. Go to this site, and follow the sequence of the articles from top to bottom. We think that you will feel confident with SocketPro after you complete the top five articles and their samples.

  2. What development languages and platforms are supported with SocketPro?

    3. So far SocketPro has fully supported the following development languages:

    • C/C++ on both windows and Linux variants

    • .NET for both C# and VB.NET on windows and Linux variants as well as window devices

    • Java on both windows and linux variants

    • Python on both window and linux variants

    • Javascript on major browsers

    • Node.js on both windows and Linux variant platforms

    • PHP on both Linux variant and windows platforms

    Note SocketPro will support more development languages as we keep on porting our core libraries onto other major platforms and creating adapters for other development languages.

    Since SocketPro fully supports HTTP and WebSocket protocols, you can access SocketPro from any other development languages or platforms through the two protocol.

  3. Can I use SocketPro with MS Visual Studio 2010 or later express for my development?

    4. Yes! You can.

  4. Does SocketPro come with free pre-compiled components?

    5. Yes! UDAParts has already created the following pre-compiled components for free to the public with open source codes. They are located at the directory ../socketpro/bin/free_services.

    • Superior performance asynchronous message persistent queue

    • High performance file streaming

    • SQL streaming for SQLite 3.11.1 or later

    • SQL streaming for MySQL 8.0.11 or later

    • SQL streaming database server plugin for MySQL 5.7.22 or older

    • SQL streaming database server plugin for Mariadb

    • SQL streaming for MySQL/Mariadb middle tier


  5. Could I use SocketPro adapter for development on JDK 1.7 or later?

    6. Yes! Java SocketPro adapter fully supports JDK 1.7 or later. However, there is no support to JDK 1.6 or earlier.

  6. Does SocketPro come with samples?

    7. SocketPro package comes with three types of samples, tutorial samples and specific-topic samples. The first type of samples is located at the directory ../socketpro/tutorials; the second at the directory ../socketpro/samples. The last types of samples are at the directory ../socketpro/src/njadapter/ for Node.js and ../socketpro/src/phpadapter/ for PHP.

    Note that the first type of samples is accompanied with short and concise articles. However, the other types of samples do not always have articles although some of them do have.

Installation and deployment questions

  1. What libraries should be distributed with my applications when I complete my project?

    You need to distribute two core libraries, and adapter libraries dependent your development languages.

    • Two core libraries. usocket for client side and uservercore for server side. Windows and devices: usocket.dll for client side and uservercore.dll for server side. Linux variants: libusocket.so for client side and libuservercore.so for server side.

    • Adapter libraries.

      • In regards to .NET for both MS and mono, distribute SproAdapter.dll for both client and server sides. The .NET adapter library can run on both windows and Linux variants. For MS window devices, use SproAdapterCe.dll instead.

      • In regards to C++, its adapter consists of files aclientw.cpp/aclientw.h (client side) and aserverw.cpp/aserverw.h (server side). There is no need to distribute extra libraries because these files are directly compiled into your codes.

      • In regards to Java, its adpater is jspa.jar. In addition, you need to distribute juclient (client) and juserver (server) jni libraries inside the directory ../socketpro/bin.

      • In regards to Python, its adapter is at the diretory ../socketpro/bin/spa.

      • In regards to PHP, its source code is located at the directory ../socketpro/src/phpadapter, but you are required to compile the code for generating libphpadapter.so (Linux) or phpadapter.dll (windows) accoding to PHP version in your web server machine.

      • In regards to Node.js, its adapter includes ../socketpro/bin/js/nja.js and Node.js platform extension library at directory ../socketpro/bin/js/node-v10.11.0.
