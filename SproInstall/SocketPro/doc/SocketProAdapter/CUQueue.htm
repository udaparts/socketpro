public class CUQueue

 

The class CUQueue is used for packing and unpacking various data into and from a memory buffer easily.

 

Step 1: In SocketPro, all of input parameters and others for a request are required to be serialized into an instance of CUQueue on client side so that we can use the method SendRequestEx to send the request to a remote SocketPro server.

 

Step 2: The buffer is retrieved into an instance of CUQueue by calling the method RetrieveBuffer when SocketPro detects a coming request on server side. Afterwards, all of original input parameters are de-serialized from the memory buffer.

 

Step 3: On server side, we could reuse the above instance of CUQueue and serialize all of returned data into the memory buffer after processing the request. At last, we call the method SendReturnData to send the return result back to a client.

 

Step 4: Finally, we use the method GetRtnBufferEx to retrieve all of return results into an instance of CUQueue when SocketPro detects a return result on client side, and de-serialize all of output data from the memory buffer.

 

With help of both SocketProAdapter and the tool uidparser.exe, all of coding the in above four steps are automatically completed. In most cases, you are not required for any modification of generated codes. However, we may need to write own code to manually pack and unpack complex data structures into and from an instance of CUQueue.

 

For details, you may refer to this short article.
