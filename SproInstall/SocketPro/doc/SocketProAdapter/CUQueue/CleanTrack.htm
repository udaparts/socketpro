public void CleanTrack()

 

Remarks

 

            When exchanging sensitive data between a client and a server, the sensitive data may be kept in a memory queue for a long time. A hacker may possibly inject some code into your application using whatever technologies. In order to eliminate such a potential security problem, you could call the function to clean the underlying memory.

 

            Refer to the native function CleanTrack.
