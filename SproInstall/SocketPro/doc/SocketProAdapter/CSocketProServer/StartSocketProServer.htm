public bool StartSocketProServer(int nPort)

public virtual bool StartSocketProServer(int nPort, int nMaxBlacklog)

 

Parameters

 

nPort

            [in]A port number for listening socket

 

nMaxBacklog

            [in]The maximum length of the queue of pending connections (default to 64)

 

Remarks

 

Calling this function starts a listening socket at the port (nPort) and prepares for accepting incoming socket connections from different clients.

 

If the function returns false, check the property LastSocketError. In most cases, the specified port number (nPort) is not available. If using SSL/TLS to secure socket communication, you may get something wrong with certificate or private key on server side.

 

Don’t confuse listening socket with peer socket. There is one and only one listening socket in a SocketPro server, which accepts many incoming connections from different clients. Each of clients has one socket on client side and one socket on server side. The socket on server side is called peer socket (CClientPeer) in SocketPro.

 

Refer to the native method StartSocketProServer.
