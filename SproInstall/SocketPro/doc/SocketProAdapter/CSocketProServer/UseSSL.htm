public static void UseSSL(USOCKETLib. tagEncryptionMethod EncryptionMethod, string strCertFile, string strPrivateKeyFile)

public static void UseSSL(USOCKETLib. tagEncryptionMethod EncryptionMethod, string strSubject, bool bMachine, bool bRoot)

public static void UseSSL(string strPfxFile, string strPassword, string strSubject, USOCKETLib.tagEncryptionMethod EncryptionMethod)

 

Parameters

 

EncryptionMethod

            [in]One of the four encryption methods, SSL23, MSSSL, TLSv1 and MSTLSv1

 

strCertFile

            [in]A string pointing to an openSSL certificate file

 

strPrivateKeyFile

            [in]A string pointing to an openSSL private key file

 

strSubject

            [in]A subject string

 

bMachine

            [in]A Boolean indicating machine (true) or current user (false) store

 

bRoot

            [in]A Boolean value, True – Trusted Root Certification Authorities; False – Personal

 

strPfxFile

            [in]A string pointing to a password protected certificate and private-key file in pfx format

 

strPassword

            [in]A password that protects the file strPfxFile

 

Remarks

 

Calling one of these functions before calling the method StartSocketProServer sets a set of parameters for securing socket communication on server side through SSL and TLS.

 

The first method is used for securing socket communication through openSSL libraries (ssleay32.dll and libeay32.dll). EncryptionMethod must be either SSL23 or TLSv1.

 

The second method is used for securing socket communication through MS SSPI. EncryptionMethod must be either MSSSL or MSTLSv1.

 

The last method is used for securing socket communication with a password protected key/certificate file in pfx format through either openSSL or MS SSPI.

 

Refer to the native methods SetCertFile, SetPrivateKeyFile, SetPfxFile and UseMSCert as well as the property DefaultEncryptionMethod
