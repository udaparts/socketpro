public class CSocketProServer

 

One instance of CSocketProServer represents one listening socket. A SocketPro server supports one listening socket only, but it manages many services. Each of services can manage many peer sockets. Each of peer sockets always corresponds to a remote client socket.

 

One instance of CSocketProServer manages one built-in instance of chat service internally. The built-in chat service supports up to 32 chat groups. SocketPro uses the built-in chat service to route a message or chat request from one client or peer socket to another client or multiple clients.

 

SocketPro supports a set of encryption methods to secure data communication between a client and a SocketPro server. To use SSL/TLS, you must set the property DefaultEncryptionMethod first before calling the method StartSocketProServer.

 

The class exposes many protected virtual (callback) functions. A few of them are often used by your code. They are OnIsPermitted, OnAccept and OnClose.

 

SocketPro supports a set of methods to authenticate a remote client from a give credential (user id and password) by overriding the virtual function OnIsPermitted. You can select an authentication method by setting the property AuthenticationMethod before calling the method StartSocketProServer.

 

SocketPro can use either window message pump or I/O completion port to drive non-blocking socket events. By default, SocketPro uses I/O completion port to drive non-blocking socket events for better performance and scalability if a window platform supports I/O completion port. On platforms win9x and me, SocketPro will use window message pump only to drive non-blocking socket events.
