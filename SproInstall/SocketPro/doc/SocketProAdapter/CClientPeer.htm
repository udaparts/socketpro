public abstract class CClientPeer

 

The class is used to manage a peer socket, which always corresponds to one remote client socket. You can use the class to process a request from one client and return a result back to a client.

 

An instance of CClientPeer-derived contains a protected instance CUQueue, m_UQueue, which is used to automatically retrieve input data of request by default. You can unpack all of request input parameters on by one. Afterwards, you can pack all of returning data by reuse of the m_UQueue and send a return result to one client by calling the method SendReturnData.

 

Therefore, an instance of CClientPeer-derived can be thought as a proxy or stub on server side.

 

In addition, the class provides a set of methods to take advantage of built-in chat service and send a message to other clients on server side.

 

The class CClientPeer exposes many virtual/abstract functions, but a few of them are often used by your code. They are OnFastRequestArrive, OnSlowRequestArrive, OnSwitchFrom, and OnReleaseResource. You may also use the virtual functions OnBaseRequestCame, OnDispatchingSlowRequest and OnSlowRequestProcessed sometimes.
