public class CSocketPool<THandler>

            where Thandler : new(),

            SocketProAdapter.ClientSide.CRequestAsynHandlerBase

 

            This generic class (or template in C++) wraps COM object USocketPool inside usocket.dll. Similar to thread pool, this COM object is used to automatically manage a few threads and a pool of client sockets and their attached asynchronous request handlers. Each of threads in a pool can host a few sockets. All of pooled sockets and their asynchronous request handlers can run in parallel.

 

            This COM object is created for simplification of middle tier and web application developments because the functions Lock and Unlock considerably reduce the complexity of managing a pool of threads and synchronizing shared asynchronous request handlers and client sockets.

 

            Once a pool of sockets is connected to a remote SocketPro server, all of other objects on both client and server sides are pooled. This way benefits the whole system performance and scalability because the whole system does not have to create and destroy various objects as well as do socket connections repeatedly. It also reduces allocated resources because all of pooled sockets and handlers are sharable by many clients.
