public class CClientSocket

 

The class CClientSocket is a wrapper of COM object USocket, which exposes a set of COM interfaces, ISocketBase, IUSocket, IUChat, and IUFast as well as IUnknown, IDispatch and IConnectionPointContainer. Although the class does not wrap all of methods of these interfaces, it indeed exposes many commonly used functions. In case you need the functions that are not wrapped in the class, you could use the method GetUSocket to get them through dotNet COM interop service.

 

Because this SocketProAdapter is written from C++/CLI, the expensive cost of COM interop service between native and manage codes is significantly reduced.

 

One instance of the class CClientSocket contains an instance of class CUQueue, which is used for sending a request to a remote SocketPro server after packing all of its input parameters. Also, the instance of CUQueue is reused to retrieve return result and unpack all of output data from a remote SocketPro server with help of an attached CRequestAsynHandlerBase-derived object. Internally, CRequestAsynHandlerBase uses the method SendRequestEx to send a request, and GetRtnBufferEx to get return result.

 

The class exposes a set of socket communication events. To monitor various socket events, you could use delegates m_OnXXXX as shown in samples and tutorials.
