[id(12), helpstring("method GetHostByName")] HRESULT GetHostByName([in]BSTR bstrHost, [in, defaultvalue(0)]VARIANT_BOOL bBlocking, [out, defaultvalue(0)]long *phHandle, [out, defaultvalue(0)]BSTR *pbstrIPAddr, [out, defaultvalue(0)]BSTR *pbstrAlias, [out,retval]BSTR *pbstrHostName);

 

Parameters

 

bstrHost

            [in]A valid remote host address string like www.udaparts.com or myservername

 

bBlocking

            [in]Asynchronously or synchronously retrieving a remote host information

 

phHanlde

            [out]Reserved

 

pbstrIPAddr

            [out]A pointer to a BSTR for receiving IP Addresses

 

pbstrAlias

            [out]A pointer to a BSTR for receiving aliases

 

pbstrHostName

            [out]A pointer to a BSTR for receiving host names

 

Remarks

           

Calling the function retrieves host information corresponding to a host name (bstrHost) like www.udaparts.com or YourMachineName from a host database either synchronously or asynchronously.

                                   

            If bBlocking is true, the function gets host information synchronously and the last three out parameters will receive host information after the function returns. Refer to winsock function gethostbyname.

 

            If bBlocking is false, the function gets host information asynchronously and the last three out parameters will not receive host information after it returns. The last three out parameters will be available when the event OnGetHostByName arrives. Refer to winsock function WSAAsyncGetHostByName.

                                   

Note that MS CE doesn't have the function WSAAsyncGetHostByName. Therefore, internally a USocket object always sets bBlocking to true.
