[id(53), helpstring("method IgnorePendingRequests")] HRESULT IgnorePendingRequests([in, defaultvalue(-1)]long lRequests, [in, defaultvalue(0)]long lStartWith);

 

Parameters

 

lRequests

            [in]The number of pending requests ignored

 

lStartWith

            [in]A zero-based index

 

Remarks

 

            Calling the function ignores an array of queued requests starting at the index (lStartWith). However, the return results of ignored requests are still processed on client side if there are indeed results returned.

 

            When sending a request in SocketPro, its request id is automatically queued into a memory buffer. When the request is processed and its result is returned, the queued request id is automatically removed from the memory buffer. In case you don’t want to return any result for the request, you may need to manually remove the queued request id by calling this method. Without removing request id from the memory, the functions WaitAll and Wait will not work correctly. Also, the size of the memory buffer may keep increasing.
