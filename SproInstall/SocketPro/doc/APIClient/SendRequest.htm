[id(38), helpstring("method SendRequest")] HRESULT SendRequest([in]short nRequestID, [in]VARIANT vtData, [in, defaultvalue(0)]VARIANT_BOOL bKeepVTFormat);

 

Parameters

 

nRequestID

            [in]A request id

 

vtData

            [in]Request input data

 

bKeepVTFormat

            [in]a boolean value indicating if the whole variant (vtData) is sent to a remote SocketPro server

 

Remarks

 

            Calling the method sends a simple request onto a remote SocketPro server. For sending complex request with different types of input data, you’d use the method SendRequestEx instead.

 

            If bKeepVTFormat is true, the function sends the whole data (vtData) in VARIANT format and SocketPro server will receive the VARIANT data structure or object in dotNet.

 

            If bKeepVTFormat is false, the function sends the data (short, int, float, double, string, ……) contained in the VARIANT structure only.

 

            Because the method requires you create a VARIANT data at run time, it is slower than SendRequestEx. Therefore, it is not recommended that you use the function to send a request unless your development environment supports automation only.
