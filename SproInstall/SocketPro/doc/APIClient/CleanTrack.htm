[id(83), helpstring("method CleanTrack")] HRESULT CleanTrack();

 

Remarks

 

                Calling this method leads cleaning all of local memory buffers first, and also sends a request to clean memories of a remote SocketPro server.

 

            When exchanging sensitive data between a client and a server, the sensitive data may be kept in memory queues for a long time. A hacker may possibly inject some code into your application using whatever technologies. In order to eliminate such a potential security problem, you could call the function to clean all of memory queues on both client and server sides. It also causes memory queues inside SocketProAdapter cleaned.

 

            SocketPro has special treatment for user password. There is no need to call the function just for cleaning password.

 

            It is considerably expensive to call the function.
