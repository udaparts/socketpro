[id(13), helpstring("method GetHostByAddr")] HRESULT GetHostByAddr([in]BSTR bstrIPAddr, [in, defaultvalue(afINet)]long lAddrFormat, [in, defaultvalue(0)]VARIANT_BOOL bBlocking, [out, defaultvalue(0)]long *phHandle, [out, defaultvalue(0)]BSTR *pbstrAlias, [out,retval]BSTR *pbstrHostName);

 

Parameters

 

bstrIPAddr

            [in]A valid remote host address string like 111.222.212.121

 

lAddrFormat

            [in]Reserved

 

bBlocking

            [in]Asynchronously or synchronously retrieving a remote host information

 

phHanlde

            [out]Reserved

 

pbstrAlias

            [out]A pointer to a BSTR for receiving alias

 

pbstrHostName

            [out]A pointer to a BSTR for receiving host name

 

Remarks

           

Calling the function retrieves host information corresponding to a host IP Address (bstrIPAddr) like 111.222.212.121 from a host database either synchronously or asynchronously.

                                   

            If bBlocking is true, the function gets host information synchronously and the last two out parameters will receive host information after the function returns. Refer to winsock function gethostbyaddr.

 

            If bBlocking is false, the function gets host information asynchronously and the last three out parameters will not receive host information after it returns. The last two out parameters will be available when the event OnGetHostByAddr arrives. Refer to winsock function WSAAsyncGetHostByAddr.

                                   

Note that MS CE doesn't have the function WSAAsyncGetHostByAddr. Therefore, internally a USocket object always sets bBlocking to true.
