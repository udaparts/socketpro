struct CSwitchInfo

{

            unsigned long m_ulSvsID;  //service id

                       

            //Call IUSocket::SetClientVersion to set the version.

            unsigned short            m_usVerMajor;

            unsigned short            m_usVerMinor;

                       

            //reserved by SocketPro. They are all 0.

            unsigned short            m_usUSockMajor;

            unsigned short            m_usUSockMinor;

 

            //m_ulParam1 and m_ulParam2 are reserved by SocketPro.

            //m_ulParam1 indicates OS information.

            //Refer to the structure OSVERSIONINFOEX and Get GetVersionEx.

            //The first byte = dwPlatformId, 2nd byte = dwMajorVersion,

            //3rd byte = dwMinorVersion, and 4th byte = dwBuildNumber.

            unsigned long m_ulParam1;

                       

            //Indicate authentication method and ping time interval in second for server param info.

            //The 1st byte = 0 and reserved for future use, 2nd byte = authentication method

            //3rd and 4th bytes contains the server ping time interval in second.

            unsigned long m_ulParam2;

                       

            //Call IUSocket::put_ClientParams to set the below three parameters

            //If you try to set five parameters, you will get uecUnexpected error.

            unsigned long m_ulParam3;

            unsigned long m_ulParam4;

                       

            //0x40000000 -- TRANSFER_SERVER_EXCEPTION of SocketProAdapter

            unsigned long m_ulParam5;

};

 

Remarks

 

            As shown in comments, the structure mainly contains the version information of both SocketPro communication protocol and service. It also contains some other useful data. At this time, m_usUSockMajor and m_usUSockMinor, which represents SocketPro communication protocol version, are 0.           

 

When a client calls the method IUSocket::SwitchTo, one structure of the client version information is sent to a SocketPro server. The SocketPro server will return its server structure back to the client too. The structure is designed so that you can easily maintain your projects as you add new features into your software components.

The structure is referenced by a set of below functions.

 

Client side:

 

            IUSocket::get/put_ClientParams

            IUSocket::GetClientUSockVersion

            IUSocket::GetClientVersion

            IUSocket::GetServerUSockVersion

            IUSocket::GetServerVersion

IUSocket::SetClientVersion

 

Server side:

 

GetClientInfo
