[id(1), helpstring("method Connect")] HRESULT Connect([in] BSTR bstrHostAddress, [in] long lHostPort, [in, defaultvalue(0)]VARIANT_BOOL bSynConnecting, [in, defaultvalue(stSTREAM)] long lSocketType, [in, defaultvalue(afINet)] long lAddrFormat, [in, defaultvalue(spIP)] long lProtocol, [in, defaultvalue(0)]long lLocalPort);

 

Parameters

 

bstrHostAddress

            [in]A valid remote host address string like www.udaparts.com, myservername or 111.222.212.121

 

lHostPort

            [in]Port which a SocketPro server is running on

 

bSynConnecting

            [in]Asynchronously or synchronously connecting to a remote host

 

lSocketType

            [in]Reserved

 

lAddrFormat

            [in]Reserved

 

lProtocol

            [in]Reserved

 

lLocalPort

            [in]Reserved

 

Remarks

           

            Calling this function starts to establish a socket connection to a remote SocketPro server with a given address (bstrHostAddress) and a given port number (lHostPort) either asynchronously or synchronously.

           

The input bstrHostAddress can’t be a null or a zero-length string.

 

No matter what the input bSynConnecting is, the socket event OnSocketConnected will happen with error code equal to 0 if the connecting is successful. If the connecting failed, the socket event OnSocketClosed will happen with a non-zero error code after the event OnSocketConnected happen.

 

Also, no matter what the input bSynConnecting is, client handshake will always be made asynchronously if SSL or TLS is used for securing socket communication.

 

Note that all the last four inputs are reserved at this time.
