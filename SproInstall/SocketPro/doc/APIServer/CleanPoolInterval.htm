void WINAPI SetCleanPoolInterval(unsigned long ulCleanPoolInterval);

unsigned long WINAPI GetCleanPoolInterval();

 

Remarks

           

            Calling the two methods sets and queries the property CleanPoolInterval. By default, it is 60,000 ms.

 

            When a peer socket is closed, its previously allocated resources are not released immediately but put into a pool instead so that a new peer socket will reuse the resources without allocating new resources. This feature is beneficial to performance especially within environment that requires fast and repeated socket connection and disconnection.

 

You will also find this feature implemented within SocketProAdapter for CClientPeer-derived instances.

 

SocketPro will remove away these resources of dead sockets for a time longer than this property. The event OnCleanPool happens at this interval.
