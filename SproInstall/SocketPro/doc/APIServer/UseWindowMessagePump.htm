bool WINAPI SetUseWindowMessagePump(bool bUseWindowMessagePump);

bool WINAPI GetUseWindowMessagePump();

 

Remarks

 

            The two functions are used for setting or querying the property UseWindowMessagePump. Starting from version 4.8.0.1, SocketPro defaults the property to false, which implies that IO completion port pump is used to drive non-blocking socket events by default with Win NT or later.

 

            You can set the property only before calling the method StartSocketProServer.

 

            The function will return false if you set the property false on Win9x and ME.
