struct CSvsContext

{

            POnIsPermitted                                 m_OnIsPermitted;    

            POnSwitchTo                                      m_OnSwitchTo;

            PonFastRequestArrive                      m_OnFastRequestArrive;    

PonBaseRequestCame                      m_OnBaseRequestCame;

            POnCleanPool                                    m_OnCleanPool;                               

            POnRequestProcessed                      m_OnRequestProcessed;     

            POnClose                                           m_OnClose;                                      

            POnReceive                                       m_OnReceive;

            POnSend                                             m_OnSend;

            enumThreadApartment                     m_enumTA;   

            PSLOW_PROCESS                          m_SlowProcess;        

            PPRETRANSLATEMESSAGE       m_PreTranslateMessage;

            PTHREAD_STARTED                     m_ThreadStarted;

            PTHREAD_DYING                          m_ThreadDying;

            POnSendReturnData                         m_OnSendReturnData;        

POnThreadCreated                           m_OnThreadCreated;

};

 

Remarks

 

            A SocketPro server manages an array of service context structures. Each of them contains a set of callback function pointers for one service. Many of callbacks are optional, but m_OnFastRequestArrive and m_SlowProcess must be set.

 

            To ease your development, SocketProAdapter is written for wrapping this structure. Therefore, you don’t have to write any code for this structure at all. Refer to CBaseService.
