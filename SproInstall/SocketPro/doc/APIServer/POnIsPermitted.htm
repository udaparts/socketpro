typedef bool (CALLBACK *POnIsPermitted) (unsigned int hSocket, unsigned long ulSvsID);

 

Parameters

 

hSocket

            [in]A peer socket handle

 

ulSvsID

            [in]A service identification number

 

Remarks

           

The callback notifies your code that a client asks for switching to the service ulSvsID. This event happens when a client calls the method IUSocket::SwitchTo on client side.

 

You use the function SetOnIsPermitted to set up your callback function. Inside the function, you can authenticate a client from its given credentials user id and password if you don’t set the property AuthenticationMethod to tagIntegrated.

 

If you do set the property AuthenticationMethod to tagIntegrated, the event still happens after SocketPro uses MS SSPI to automatically authenticate a client. However, giving the final permission does not depend on your return, true or false.

 

This event always happens within main thread.
