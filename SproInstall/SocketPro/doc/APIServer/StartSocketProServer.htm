bool WINAPI StartSocketProServer(unsigned int uiPort, unsigned int uiMaxBacklog = 64);

 

Parameters

 

uiPort

            [in]A port running a listening socket

 

uiMaxBacklog

            [in]The maximum length of the queue of pending connections

 

Remarks

           

            Calling this function starts listening for incoming socket connections from clients at a given port (uiPort) with maximum length of the queue of pending connections (uiMaxBacklog).

           

            Note that listening socket is bound to all of network cards available within a server machine. By default, the queue of pending connection is 64, but the size of the queue is 5 with many non-server machines.

 

            If the function is failed, the function returns false and there are a few possible causes below.

 

            a). The given port (uiPort) is already owned by other socket server application so that it is not available any more.

            b). The previous specified certificate, key or their files are wrong, if you use SSL or TLS to secure your socket connections.

 

            You can use window system function GetLastError to query error code.

 

            Correspondingly, you should use the function StopSocketProServer to stop the listening socket before calling the function UninitSocketProServer, if you don’t need the SocketPro server any more.

 

            Also, refer to the function PostQuitPump.
